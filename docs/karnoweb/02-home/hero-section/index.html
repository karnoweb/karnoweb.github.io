سکشن هیرو (Hero Section)

وضعیت: پیش‌نویس
بخش: 02-home
آخرین به‌روزرسانی: 2026-07-01

Goal (هدف)​

کاربر باید در ۳ تا ۵ ثانیه بفهمد:

  • کارنووب دقیقاً چه می‌سازد (سیستم رشد + زیرساخت دیجیتال)
  • این کار چه ارزشی برای کسب‌وکار دارد (فروش/عملیات/مقیاس)
  • قدم بعدی چیست (رزرو جلسه / درخواست دمو)

Target Audience (مخاطب هدف)​

  • مدیر/مالک کسب‌وکار (به دنبال نتیجه و مسیر شفاف)
  • مدیر مارکتینگ/فروش (به دنبال لید و رشد قابل سنجش)
  • مدیر IT/CTO (به دنبال معماری و قابلیت اتکا)

User Problems (مسئله‌های کاربر)​

  • ابهام در تعریف کارنووب (آژانس؟ تیم توسعه؟ محصول؟)
  • بی‌اعتمادی به وعده‌های کلی
  • ترس از شروع (ریسک/زمان/هزینه)

Solutions (راه‌حل)​

  • پیام یک‌خطی Outcome محور + زیرتیتر کوتاه
  • CTA اصلی Warm + CTA ثانویه کاهش‌ریسک
  • نمایش «محصول/داشبورد واقعی» به‌عنوان اثبات فنی
  • Micro trust زیر CTAها

CTA (دعوت به اقدام)​

Primary (Warm — مخصوص صفحات مارکتینگ)​

  • متن پیشنهادی: «رزرو جلسه استراتژی (رایگان)»
  • لینک: /contact?type=consultation&source=home-hero

Secondary (Outline)​

  • متن پیشنهادی: «مشاهده نمونه‌کارها»
  • لینک: /portfolio

Soft CTA (اختیاری)​

  • متن پیشنهادی: «درخواست دمو محصولات»
  • لینک: /contact?type=demo&source=home-hero

SEO Keywords (کلیدواژه‌ها)​

  • زیرساخت دیجیتال
  • سیستم رشد
  • اتوماسیون کسب‌وکار
  • ERP / CRM (به‌صورت طبیعی در زیرتیتر/ویژوال)

UI Ideas (ایده‌های UI)​

ساختار (Layout)​

  • Desktop: دو ستونه (متن + ویژوال)
  • Mobile: تک‌ستونه (متن → CTAها → ویژوال → Trust)

پس‌زمینه​

  • Dark Hero با توکن --hero-bg (Glow کنترل‌شده)
  • ممنوع: بک‌گراندهای شلوغ/نئون/پرتحرک

ویژوال (به ترتیب اولویت)​

  1. اسکرین‌شات واقعی داشبورد یکی از محصولات کارنووب
  2. اسکرین‌شات واقعی پنل گزارش/اتوماسیون
  3. فلو/نقشه اتوماسیون (Diagram سبک)

ممنوع:

  • عکس استاک (جلسه/دست تایپ/…)
  • تصویر انتزاعی بدون مفهوم

متن و کپی (Copy)​

Badge (بالای تیتر):

  • «شریک رشد دیجیتال کسب‌وکارها»

تیترهای پیشنهادی (یکی انتخاب شود):

  1. «زیرساخت دیجیتال کسب‌وکار شما را می‌سازیم.»
  2. «از ایده تا رشد — سیستم‌های مقیاس‌پذیر برای کسب‌وکارهای مدرن.»
  3. «وب‌سایت، اتوماسیون و سامانه‌های سازمانی — یکپارچه و قابل رشد.»

زیرتیتر (حداکثر ۲ خط):

  • «کارنووب با ترکیب توسعه نرم‌افزار، سئو تکنیکال، اتوماسیون و طراحی تجربه کاربری، سیستم‌هایی می‌سازد که فروش و عملیات را سریع‌تر و دقیق‌تر می‌کند.»

Micro Trust زیر CTAها (۴ مورد کوتاه)​

  • «معماری مقیاس‌پذیر»
  • «API-محور (API-First)»
  • «بهینه برای سئو (SEO-Ready)»
  • «پشتیبانی واقعی»

Sections (اجزای سکشن)​

  • Announcement Bar (اختیاری)
  • Badge / Eyebrow
  • Headline
  • Subheadline
  • CTA Row (Primary + Secondary)
  • Micro Trust Row
  • Product Visual (Dashboard Screenshot)
  • (اختیاری) Live Metrics کوچک

Components (کامپوننت‌ها)​

  • AnnouncementBar
  • HeroBadge
  • HeroCTAGroup
  • MicroTrustList
  • ProductScreenshotFrame
  • MetricMiniCards (اختیاری)

Animations (انیمیشن‌ها)​

  • ورود عناصر با Fade + translateY کوچک (سبک)
  • Hover برای CTA و کارت ویژوال
  • Floating بسیار ملایم برای 1–2 کارت (اگر عملکرد قربانی نشود)
  • رعایت prefers-reduced-motion

Conversion Strategy (استراتژی تبدیل)​

  • Primary را «رزرو جلسه» نگه می‌داریم (کم‌ریسک + قدم بعدی روشن)
  • ویژوال واقعی = کاهش شک و افزایش کلیک CTA
  • Micro trust باید قبل از اسکرول، ترس‌های رایج را کم کند

Notes (یادداشت‌ها)​

  • رنگ CTA اصلی در Hero باید Warm باشد (طبق 01-global/cta-strategy.md)
  • از HEX مستقیم در CSS خودداری شود؛ از توکن‌های 00-brand/colors-and-theme.md استفاده شود.
