# سکشن اثبات اجتماعی (Social Proof)


## Goal (هدف)

- ایجاد اعتماد سریع بلافاصله بعد از Hero
- کاهش شک کاربر نسبت به «واقعی بودن» توان فنی/تجربه
- افزایش کلیک روی CTAهای سکشن‌های بعدی

## Target Audience (مخاطب هدف)

- کاربر تصمیم‌گیر (مدیرعامل/مدیر مارکتینگ/مدیر عملیات)
- کاربر فنی (CTO/IT Manager)

## User Problems (مسئله‌های کاربر)

- «نمونه واقعی دارید؟»
- «این حرف‌ها ادعایی نیست؟»
- «شرکت‌های مشابه ما هم با شما کار کرده‌اند؟»

## Solutions (راه‌حل)

ترکیب ۳ لایه اعتماد (به ترتیب اهمیت):
1) لوگو مشتریان / صنایع
2) اعداد و شاخص‌ها (Metrics)
3) نقل‌قول کوتاه (Testimonial snippet)

## CTA (دعوت به اقدام)

- Soft CTA: «مشاهده نمونه‌کارها» → `/portfolio`  
(CTA این سکشن باید نرم باشد؛ Primary صفحه همان «رزرو جلسه» می‌ماند.)

## SEO Keywords (کلیدواژه‌ها)

- (در این سکشن تمرکز روی تبدیل است؛ سئو مستقیم اولویت ندارد.)

## UI Ideas (ایده‌های UI)

### چینش پیشنهادی (۳ ردیف)

**ردیف ۱ — Trust Bar**
- «مورد اعتماد تیم‌های رشد و عملیات»
- ردیف لوگوها (حداکثر 8–12 لوگو)
- اگر اجازه لوگو نداریم: به‌جای لوگو از «برچسب صنعت» استفاده شود:
  - خرده‌فروشی
  - آموزش
  - خدمات
  - تولید
  - سلامت
  - …

**ردیف ۲ — Metrics**
- 3 تا 5 عدد کلیدی (Counter)
- مثال ساختار (قابل تغییر با داده واقعی):
  - «+120 پروژه تحویل‌شده»
  - «+40 مشتری فعال»
  - «99.9٪ پایداری سرویس (SLA)»
  - «زمان پاسخ‌گویی < 24 ساعت»

**ردیف ۳ — نقل‌قول کوتاه**
- یک کارت کوچک با امتیاز 5 ستاره + جمله 1 خطی
- اگر نام مشتری قابل ذکر نیست: «مدیر عملیات — صنعت خرده‌فروشی»

### سبک
- پس‌زمینه روشن و ساده (برای استراحت چشم بعد از Hero تیره)
- Badgeها و خطوط جداکننده با رنگ‌های Cool
- استفاده محدود از Warm (فقط اگر لازم شد روی یک Badge مثل «پیشنهاد ویژه»)

## Sections (اجزای سکشن)

- Section Header (Title + Subtitle کوتاه)
- Logos / Industries Row
- Metrics Row (Counter)
- Testimonial Snippet (اختیاری)

## Components (کامپوننت‌ها)

- LogoMarquee (یا Grid لوگو)
- IndustryChips (Fallback)
- MetricCard (با tabular-nums)
- TestimonialMiniCard

## Animations (انیمیشن‌ها)

- Logo row: اسکرول افقی بسیار ملایم (اختیاری) یا ثابت
- Counter animation هنگام ورود به viewport
- Hover بسیار ملایم روی MetricCard

(طبق `01-global/animations.md`)

## Conversion Strategy (استراتژی تبدیل)

- این سکشن باید «شک» را کم کند تا کاربر:
  - خدمات/محصولات را جدی بگیرد
  - پایین‌تر اسکرول کند
  - در نهایت CTA اصلی را بزند

## Notes (یادداشت‌ها)

- هر عدد باید منبع داخلی داشته باشد (Dashboard/CRM/گزارش پروژه). اگر نداریم:
  - فعلاً از اعداد استفاده نکنید یا با برچسب «حداقلی/محافظه‌کارانه» وارد کنید.
- اگر لوگوها مجوز ندارند، از «صنعت + نوع پروژه» استفاده شود.
