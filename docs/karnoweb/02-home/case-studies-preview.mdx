## سکشن Case Studies

### هدف
**اثبات واقعی** توانایی کارنووب — نه ادعا.
«قبلاً موفق بوده‌اند» قوی‌ترین عامل تصمیم‌گیری است.

### پیام اصلی
> «کارنووب فقط درباره رشد صحبت نمی‌کند — نتایج واقعی ایجاد می‌کند.»

---

### ساختار سکشن

**Header:**

| عنصر | محتوا |
|------|--------|
| **Title** | نتایجی که واقعاً اتفاق افتاده |
| **Subtitle** | پروژه‌های واقعی — اعداد واقعی |

---

**ساختار هر Case Study کارت:**
```
┌─────────────────────────────────────────┐
│  [تصویر پروژه]                         │
│  [Tag: صنعت] [Tag: خدمت]             │
│                                         │
│  نام کسب‌وکار / صنعت                  │
│                                         │
│  مشکل (یک جمله):                      │
│  «موجودی ۳ شعبه به‌صورت دستی مدیریت» │
│                                         │
│  راهکار: Shop ERP + اتوماسیون          │
│                                         │
│  نتایج:                                │
│  ↑ ۲۸۰٪  دقت موجودی                 │
│  ↓ ۶۰٪   زمان پردازش سفارش          │
│                                         │
│  [مشاهده جزئیات کامل →]              │
└─────────────────────────────────────────┘
```

**تعداد:** ۳ Case Study از ۳ حوزه مختلف (سایت، سئو، ERP)

**قوانین محتوا:**
- عدد داشته باشد — «بهتر شد» کافی نیست
- مشکل واقعی باشد — نه کلیشه
- نام صنعت (نه لزوماً نام شرکت)
- دکمه «مشاهده جزئیات» به صفحه Portfolio

---

**CTAها:**

| | متن | لینک |
|-|-----|------|
| **Primary** | مشاهده کامل نمونه‌کارها | `/portfolio` |
| **Secondary** | شروع پروژه مشابه | `/contact?source=home-casestudy` |

---

**UI Direction:**
- Desktop: Grid 3 ستونه
- کارت‌های بزرگ با تصویر بالا
- Before/After برای پروژه‌های طراحی
- Charts/Metrics برای پروژه‌های SEO و ERP

---

**حس نهایی:**
> «این‌ها قبلاً پروژه‌های واقعی موفق انجام داده‌اند.»
