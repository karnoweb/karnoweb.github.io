استراتژی CTA در کل سایت

هدف​

CTA در کارنووب «فقط کلیک گرفتن» نیست؛ هدف این است که کاربر وارد مسیر تبدیل شود:

بازدیدکننده → لید → جلسه/دمو → پروژه/اشتراک → نگهداشت → فروش تکمیلی

هر CTA باید دقیقاً مشخص کند:

  • کاربر الان کجای مسیر است؟
  • قدم بعدی چیست؟
  • چرا همین حالا ارزش انجام دارد؟

۳ نیت اصلی کاربر = ۳ نوع CTA​

1) نیت اطلاعاتی (آگاهی)​

کاربر در حال تحقیق است و هنوز آماده خرید نیست.

CTA مناسبمثال متناستایل پیشنهادی
دریافت محتوای رایگان«دریافت چک‌لیست سئو رایگان»Soft (Outline/Ghost)
مطالعه بیشتر«مشاهده راهنمای کامل»لینک ساده
عضویت«عضویت در خبرنامه»فرم ساده

2) نیت مقایسه‌ای (ارزیابی)​

کاربر گزینه‌ها را بررسی می‌کند و دنبال شواهد است.

CTA مناسبمثال متناستایل پیشنهادی
مشاهده دمو«درخواست دمو ERP»Secondary (Outline)
رزرو مشاوره«رزرو جلسه مشاوره رایگان»Secondary
مقایسه پلن‌ها«مقایسه پلن‌ها»لینک با آیکون

3) نیت تصمیم (آماده اقدام)​

کاربر آماده است و فقط یک تریگر می‌خواهد.

CTA مناسبمثال متناستایل پیشنهادی
شروع همکاری«شروع پروژه»Primary
رزرو جلسه«رزرو جلسه استراتژی»Primary
دریافت پیشنهاد قیمت«دریافت پیشنهاد قیمت»Primary

سیستم ۳ لایه CTA در هر صفحه (قاعده ثابت)​

هر صفحه باید دقیقاً این ۳ لایه را داشته باشد:

1) CTA اصلی (Primary) — فقط یک عدد​

  • پررنگ‌ترین و مهم‌ترین
  • متن: فعل + نتیجه

قاعده رنگ (طبق استاندارد جدید رنگ‌ها):

  • صفحات مارکتینگ (Home/Services/Products/Portfolio/Blog): Primary = Warm (Action Orange / --gradient-warm)
  • پنل‌ها/داشبوردها (Client Panel): Primary = Cool (Brand Blue / --gradient-cool)

2) CTA ثانویه (Secondary) — ۱ تا ۲ عدد​

  • برای کاربرانی که هنوز آماده نیستند
  • معمولاً Outline

3) CTA نرم (Soft) — در طول صفحه​

  • لینک، Badge، یا دانلود
  • هدف: گرفتن لید بدون فشار

CTA به تفکیک صفحه (پیشنهادی)​

صفحهCTA اصلی (Primary)CTA ثانویه (Secondary)CTA نرم (Soft)
Home«رزرو جلسه استراتژی»«مشاهده محصولات»«دریافت راهنمای رشد»
Services«رزرو مشاوره رایگان»«مشاهده نمونه‌کار مرتبط»«دریافت برآورد اولیه»
Products«درخواست دمو»«مقایسه پلن‌ها»«دانلود بروشور»
Blog«دریافت چک‌لیست رایگان»«مشاهده خدمات مرتبط»«عضویت در خبرنامه»
Portfolio«شروع پروژه مشابه»«مشاهده پروژه‌های دیگر»«دانلود گزارش خلاصه»
About«همکاری با ما»«مشاهده خدمات»«تماس با تیم»
Contact«ارسال درخواست»«رزرو جلسه»«واتساپ»

استاندارد UI دکمه‌ها (Token-first)​

در کد واقعی، HEX مستقیم ننویسید. از توکن‌ها استفاده کنید.

Primary CTA (Marketing / Warm)​

.btn-primary-warm {
  background: var(--gradient-warm);
  color: #fff;
  border-radius: var(--radius-md);
  padding: 14px 28px;
  font-weight: 600;
  min-width: 160px;
  box-shadow: var(--shadow-brand-warm);
  transition: transform 220ms cubic-bezier(0.2, 0.8, 0.2, 1),
              box-shadow 220ms cubic-bezier(0.2, 0.8, 0.2, 1);
}

.btn-primary-warm:hover {
  transform: translateY(-1px);
}

Primary (Product / Cool)​

.btn-primary-cool {
  background: var(--gradient-cool);
  color: #fff;
  border-radius: var(--radius-md);
  padding: 14px 28px;
  font-weight: 600;
  min-width: 160px;
  box-shadow: var(--shadow-brand-cool);
  transition: transform 220ms cubic-bezier(0.2, 0.8, 0.2, 1),
              box-shadow 220ms cubic-bezier(0.2, 0.8, 0.2, 1);
}

Secondary (Outline)​

.btn-secondary {
  background: transparent;
  color: inherit;
  border: 1.5px solid rgba(203, 213, 225, 1);
  border-radius: var(--radius-md);
  padding: 13px 27px;
}

Soft CTA (Link)​

.cta-link {
  color: inherit;
  text-decoration: none;
  border-bottom: 1px solid transparent;
  transition: border-color 220ms ease;
}

.cta-link:hover {
  border-bottom-color: currentColor;
}

فرمول‌های متن CTA (Copywriting)​

فرمول ۱ — فعل + نتیجه​

  • «رشد فروش را شروع کنید»
  • «درخواست دمو و بررسی نیاز»
  • «تحلیل رایگان سایت را دریافت کنید»

فرمول ۲ — مشکل + راه‌حل​

  • «سایتتان مشتری جذب نمی‌کند؟ تحلیل رایگان بگیرید»
  • «فرآیندها کند است؟ دمو ERP را ببینید»

فرمول ۳ — ارزش رایگان + اقدام ساده​

  • «دریافت چک‌لیست سئو (رایگان)»
  • «بررسی اولیه زیرساخت (۲ دقیقه)»

اشتباهات رایج (ممنوع)​

اشتباهچرا بد استجایگزین
«بیشتر بدانید»مبهم«مشاهده نمونه‌کار سئو»
«کلیک کنید»ضعیف«دریافت تحلیل رایگان»
