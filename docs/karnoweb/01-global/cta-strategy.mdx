## سیستم CTA در کل سایت

### هدف سیستم CTA

هدف CTA در کارنووب **فقط کلیک گرفتن نیست** — هدف اصلی یک مسیر تبدیل مشخص است:

```
بازدیدکننده  →  لید  →  جلسه مشاوره  →  پروژه  →  Upsell محصول
```

هر CTA باید مشخص باشد **این کاربر الان کجای این مسیر است** و قدم بعدی را روشن نشان دهد.

---

### سه نوع نیت کاربر = سه نوع CTA

**نوع ۱ — Awareness (اطلاعاتی)**
کاربر در حال تحقیق است. هنوز آماده خرید نیست.

| CTA مناسب | مثال متن | رنگ/استایل |
|-----------|---------|------------|
| دانلود محتوای رایگان | «دریافت چک‌لیست سئو رایگان» | Soft — Outline یا Ghost |
| مطالعه بیشتر | «مشاهده راهنمای کامل» | لینک ساده |
| ثبت ایمیل | «عضویت در خبرنامه» | فرم ساده |

**نوع ۲ — Consideration (مقایسه‌ای)**
کاربر دارد گزینه‌ها را بررسی می‌کند.

| CTA مناسب | مثال متن | رنگ/استایل |
|-----------|---------|------------|
| مشاهده دمو | «مشاهده دمو زنده ERP» | Secondary — Outline رنگی |
| دریافت مشاوره | «رزرو جلسه مشاوره رایگان» | Secondary |
| مقایسه | «مقایسه پلن‌ها» | لینک با آیکون |

**نوع ۳ — Decision (تصمیم)**
کاربر آماده است، فقط نیاز به تریگر دارد.

| CTA مناسب | مثال متن | رنگ/استایل |
|-----------|---------|------------|
| شروع پروژه | «شروع پروژه» | Primary — پر، Bold |
| رزرو جلسه | «رزرو جلسه استراتژی» | Primary |
| دریافت قیمت | «دریافت پیشنهاد قیمت» | Primary |

---

### سیستم ۳ لایه CTA در هر صفحه

هر صفحه باید دقیقاً سه لایه CTA داشته باشد — نه بیشتر، نه کمتر:

**Primary CTA — یکی در هر صفحه**
- پررنگ‌ترین، بزرگ‌ترین، مهم‌ترین
- رنگ: `Brand Blue #1A56DB`
- متن: فعل قوی + نتیجه / «رزرو جلسه مشاوره» / «درخواست دمو»

**Secondary CTA — یکی یا دوتا**
- کمتر از Primary، اما هنوز مشخص
- استایل: Outline با رنگ برند
- برای کاربری که هنوز آماده نیست

**Soft CTA — پراکنده در صفحه**
- لینک ساده یا Badge
- برای لیدهای اطلاعاتی: دانلود، مطالعه، عضویت
- هدف: وارد کردن به Funnel بدون فشار

### قوانین طراحی دکمه CTA

**Primary Button:**
```css
background: #1A56DB;
color: #FFFFFF;
padding: 14px 28px;
border-radius: 8px;
font-weight: 600;
font-size: 16px;
min-width: 160px;
box-shadow: 0 4px 14px rgba(26, 86, 219, 0.4);
transition: all 0.2s ease;

/* Hover */
background: #1649C0;
box-shadow: 0 6px 20px rgba(26, 86, 219, 0.5);
transform: translateY(-1px);
```

**Secondary Button:**
```css
background: transparent;
color: #1A56DB;
border: 1.5px solid #1A56DB;
padding: 13px 27px;
/* سایر properties مثل Primary */

/* Hover */
background: rgba(26, 86, 219, 0.06);
```

**Soft CTA (لینک):**
```css
color: #1A56DB;
text-decoration: none;
border-bottom: 1px solid transparent;
transition: border-color 0.2s;

/* Hover */
border-bottom-color: #1A56DB;
```

### فرمول‌های کپی‌رایتینگ CTA

**فرمول ۱ — Action + Outcome:**
«[فعل] + [نتیجه]»
- «رشد کسب‌وکار خود را شروع کنید»
- «سیستم فروشتان را هوشمند کنید»
- «دمو ERP را مشاهده کنید»

**فرمول ۲ — Problem + Solution:**
«[مشکل]؟ [راه‌حل]»
- «سایتتان مشتری جذب نمی‌کند؟ تحلیل رایگان بگیرید»
- «فرآیندهایتان کند است؟ ERP را ببینید»

**فرمول ۳ — Value First:**
«[ارزش رایگان] + [اقدام ساده]»
- «دریافت استراتژی رشد رایگان»
- «تحلیل رایگان سئوی سایت — ۲ دقیقه»

### اشتباهات رایج CTA (باید اجتناب شود)

| اشتباه | چرا | جایگزین |
|---------|-----|---------|
| «بیشتر بدانید» | مبهم، هدف ندارد | «مشاهده نمونه‌کارهای سئو» |
| «کلیک کنید» | ضعیف | «دریافت تحلیل رایگان» |
| ۴+ CTA در یک سکشن | سردرگمی | حداکثر ۲ CTA اصلی |
| CTA یکسان در همه صفحات | بی‌ربط | CTA بر اساس نیت صفحه |
| بدون Trust Signal | کاهش Conversion | همیشه زیر CTA: ✓ رایگان ✓ پاسخ سریع |
