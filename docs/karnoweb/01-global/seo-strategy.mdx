# استراتژی سئوی سراسری (SEO Strategy)

این سند قوانین سئوی «مشترک بین همه صفحات» را مشخص می‌کند:
- استاندارد متادیتا
- ساختار هدینگ‌ها
- URL
- تصاویر
- اسکیما
- اهداف Core Web Vitals

---

## قوانین فنی (برای همه صفحات)

### Title Tag
- فرمت: `[کلمه کلیدی اصلی] | [توضیح کوتاه] | کارنووب`
- حداکثر 60 کاراکتر
- کلمه کلیدی نزدیک ابتدای عنوان باشد

### Meta Description
- فرمت: [نیاز/مسئله] + [راه‌حل] + [CTA]
- 140–155 کاراکتر
- برای هر صفحه یکتا

### ساختار Heading
- `H1`: فقط یک عدد در هر صفحه (شامل کلمه کلیدی اصلی)
- `H2`: بخش‌های اصلی
- `H3`: زیربخش‌ها
- هدینگ برای «استایل» استفاده نشود

### ساختار URL
- فارسی در URL ممنوع (برای استاندارد و اشتراک‌گذاری)
- مثال: `/services/seo`
- استفاده از `-` (hyphen)، نه `_`
- عمق مسیر: حداکثر 3 سطح

### سئوی تصاویر
- `alt` برای همه تصاویر (فارسی + نزدیک به کلمه کلیدی)
- فرمت: WebP/AVIF
- حجم تصاویر معمولی: ترجیحاً زیر 100KB
- Lazy-load برای همه به جز تصویر LCP

---

## Schema Markup (ساختاریافته)

- Organization: در کل سایت
- BreadcrumbList: همه صفحات داخلی
- Service: صفحات خدمات
- SoftwareApplication: صفحات محصولات
- FAQPage: هر جا FAQ داریم
- HowTo: صفحات فرآیند/راهنمای مرحله‌ای
- Person: صفحات تیم (در صورت داشتن)

---

## Core Web Vitals (هدف)

- LCP: زیر 2.5 ثانیه
- INP: زیر 100ms
- CLS: زیر 0.1

---

## قوانین محتوایی (Content SEO)

- هر صفحه خدمت باید:
  - مسئله → راه‌حل → فرآیند → نمونه‌کار → FAQ → CTA
- هر صفحه محصول باید:
  - Use case → ویژگی‌ها → امنیت/اعتماد → پلن‌ها → دمو → FAQ

---

## لینک‌سازی داخلی (Internal Linking)

- از Blog به Service/Product مرتبط
- از Service به Case Study مرتبط
- از Product به ابزار/منابع آموزشی مرتبط
- Breadcrumb همیشه فعال
