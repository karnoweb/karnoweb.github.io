## صفحه خدمات UI/UX

### پیام اصلی
> «طراحی خوب آن چیزی نیست که فقط زیبا باشد — آن چیزی است که می‌فروشد.»

---

### ساختار کامل صفحه

#### سکشن ۱ — Hero

| عنصر | محتوا |
|------|--------|
| **تیتر** | طراحی تجربه کاربری که نرخ تبدیل را بالا می‌برد |
| **زیرتیتر** | هر پیکسل بر اساس داده رفتار کاربر — نه سلیقه طراح |
| **CTA اول** | دریافت مشاوره طراحی → `/contact?service=uiux` |
| **ویژوال** | Before/After یک صفحه واقعی با نمایش آمار بهبود |

**آمار زیر Hero:**
```
+۴۵٪ میانگین بهبود Conversion Rate | ۳ ثانیه: حداکثر زمان برای جذب کاربر | ۶۰٪+ ترافیک از موبایل
```

---

#### سکشن ۲ — مشکلات رایج

| مشکل | نشانه |
|------|--------|
| نرخ تبدیل پایین | بازدیدکننده دارید اما مشتری نه |
| خروج سریع کاربر | Bounce Rate بالای ۷۰٪ |
| سردرگمی در مسیر خرید | کاربر نمی‌داند بعد چه کاری کند |
| تجربه موبایل ضعیف | سایت روی موبایل کار می‌کند اما راحت نیست |
| طراحی قدیمی | ظاهر سایت اعتماد ایجاد نمی‌کند |

---

#### سکشن ۳ — خدمات UI/UX

| خدمت | شامل |
|------|-------|
| **User Research** | مصاحبه با کاربران، بررسی Heatmap، تحلیل رفتار |
| **Wireframe & Prototype** | طراحی مسیر کاربر، تست قبل از کد |
| **UI Design** | طراحی رابط زیبا و کاربردی در Figma |
| **Design System** | کتابخانه کامپوننت برای توسعه‌دهنده |
| **Usability Testing** | تست با کاربران واقعی |
| **Redesign** | بازطراحی سایت یا اپ موجود با هدف Conversion |

---

#### سکشن ۴ — فرآیند

```
[Research]  →  [Define]  →  [Design]  →  [Test]  →  [Deliver]
کاربرشناسی    مسئله‌تعریف   طراحی       آزمون       تحویل به Dev
```

---

#### سکشن ۵ — CTA نهایی

**Title:** بگذار کاربرانت بگویند «این سایت راحته!»

| CTA | متن | لینک |
|-----|-----|------|
| Primary | دریافت مشاوره طراحی | `/contact?service=uiux` |
| Secondary | مشاهده نمونه‌کارها | `/portfolio?filter=design` |

### ارتباط با صفحات دیگر
- `←` از: `/services`، صفحه طراحی سایت، صفحه اپلیکیشن
- `→` به: `/contact`، `/services/website`، `/services/app`
- **Bundle Offer:** «سایت اختصاصی + UI/UX = بهترین نتیجه» → لینک به `/contact?bundle=website-uiux`
