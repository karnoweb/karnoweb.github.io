# بلاگ کارنووب — نمای کلی (Blog Overview)


## Goal (هدف)

بلاگ کارنووب سه نقش هم‌زمان دارد:
1) **سئو و ترافیک ارگانیک** (بلندمدت و پایدار)
2) **اعتبارسازی و نمایش تخصص** (Trust / Authority)
3) **ورودی قیف تبدیل** (هر مقاله باید یک «قدم بعدی» داشته باشد)

بلاگ نباید فقط «محتوا برای محتوا» تولید کند؛ هر مقاله باید مشکل واقعی کاربر هدف را حل کند و او را یک قدم به تصمیم نزدیک‌تر کند.

## Target Audience (مخاطب هدف)

- مدیر/مالک کسب‌وکار (دنبال رشد و کنترل)
- مدیر مارکتینگ/فروش (دنبال لید و نرخ تبدیل)
- مدیر IT/CTO (دنبال معماری و ابزارهای درست)
- استارتاپ و کارآفرین (دنبال مسیر و جلوگیری از اشتباه)

## User Problems (مسئله‌های کاربر)

- «نمی‌دانم از کجا شروع کنم» (سئو/ERP/CRM/اتوماسیون)
- «نمی‌دانم چقدر باید هزینه کنم»
- «ابزارها را می‌شناسم ولی نمی‌دانم چطور انتخاب کنم»
- «نمونه‌های واقعی و نتیجه می‌خواهم، نه نظریه»

## Solutions (راه‌حل)

- محتوای نتیجه‌محور (Outcome-first)
- راهنماهای کاربردی (How-to / Checklist)
- مقایسه ابزارها/رویکردها
- Case Study و تجربیات واقعی
- محتوای Evergreen برای سئوی بلندمدت

## CTA (دعوت به اقدام)

### صفحه اصلی بلاگ
- CTA اصلی: «عضویت در خبرنامه» یا «دریافت بهترین مقالات»
- CTA ثانویه: «مشاوره رایگان» (Sticky یا در Sidebar)

### داخل مقالات
- CTA مرتبط با موضوع مقاله (خدمت/محصول/Lead Magnet)
- (جزئیات: `05-blog/cta-strategy.md`)

## SEO Keywords (کلیدواژه‌های کلی بلاگ)

- آموزش سئو
- پیاده‌سازی ERP و CRM
- اتوماسیون کسب‌وکار
- طراحی سایت اختصاصی
- رشد دیجیتال
- بازاریابی محتوا

## UI Ideas (ایده‌های UI) — Token-first

- صفحه اصلی بلاگ:
  - Featured Article (بزرگ)
  - Grid مقالات (3 ستون Desktop / 2 Tablet / 1 Mobile)
  - Filter/Tab دسته‌بندی
  - Sidebar: «مقالات پربازدید» + CTA

- صفحه مقاله:
  - Typography خوانا (Body 18px / Line-height 1.75)
  - Sticky Table of Contents (در Desktop)
  - Progress bar (اسکرول)
  - Contextual CTAها
  - مقالات مرتبط در انتها

## Conversion Strategy (استراتژی تبدیل)

```txt
کاربر از سئو/شبکه اجتماعی وارد مقاله می‌شود
         ↓
مشکلش را می‌بیند + راه‌حل می‌گیرد (Trust)
         ↓
یک Lead Magnet مرتبط دریافت می‌کند (ایمیل/شماره)
         ↓
وارد CRM می‌شود + سکانس Nurture شروع می‌شود
         ↓
هدایت به صفحه خدمت/محصول مرتبط
         ↓
رزرو جلسه / درخواست دمو
```

## Performance Targets

- LCP: زیر 2.5 ثانیه
- تصاویر: WebP + Lazy-load (به‌جز featured image)
- فهرست مطالب برای مقالات بالای 1500 کلمه

## Notes (یادداشت‌ها)

### Schema
- `Article` یا `BlogPosting` برای همه مقالات
- `BreadcrumbList` برای همه صفحات داخلی بلاگ

### Canonical
- اگر مقاله‌ای در جای دیگر هم منتشر شده، Canonical مشخص شود.
