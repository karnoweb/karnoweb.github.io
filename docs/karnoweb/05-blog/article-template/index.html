قالب استاندارد مقاله (Article Template)

هدف این قالب​

  • یکپارچگی ساختاری همه مقالات بلاگ
  • کاهش زمان نگارش و ادیت
  • تضمین سئو On-page در همه مقالات
  • تضمین وجود «قدم بعدی» (CTA)

قبل از نوشتن (Pre-writing Checklist)​

  • کلیدواژه اصلی مشخص شده؟
  • کلیدواژه‌های مرتبط و LSI مشخص شده؟
  • هدف مقاله مشخص است؟ (Evergreen / Authority / Conversion)
  • Lead Magnet مرتبط تعریف شده؟
  • CTA داخل مقاله مشخص است؟
  • لینک‌های داخلی (به خدمات/محصولات/مقالات) شناسایی شده؟
  • طول پیشنهادی تعیین شده؟ (800 / 1500 / 2500+ کلمه)

ساختار استاندارد مقاله​

1) Metadata (پشت صحنه)​

فیلدمقدار
Title Tag[کلیدواژه اصلی] — [توضیح]
Meta Description[مشکل] + [راه‌حل] + [CTA] — ۱۴۰–۱۵۵ کاراکتر
Canonical URLآدرس کامل صفحه
دسته‌بندییکی از دسته‌های /blog/[category]
تگ‌هاحداکثر ۵ تگ مرتبط
نویسندهنام + لینک پروفایل
تاریخ انتشار—
تاریخ آخرین به‌روزرسانی—
Open Graph Image۱۲۰۰×۶۳۰ پیکسل (WebP/JPG)
SchemaArticle + BreadcrumbList + FAQPage (در صورت وجود)

2) Featured Image​

  • اندازه: ۱۲۰۰×۶۳۰ پیکسل
  • فرمت: WebP
  • Alt text: شامل کلیدواژه اصلی
  • ممنوع: عکس استاک عمومی (جلسه/دست/لپ‌تاپ بی‌ربط)
  • پیشنهاد: Screenshot واقعی / Diagram / Infographic ساده

3) Title (H1)​

  • حاوی کلیدواژه اصلی (ترجیحاً نزدیک ابتدا)
  • جذاب و مسئله‌محور
  • حداکثر ۶۰ کاراکتر برای نمایش کامل در Google

الگوهای عنوان:

  • «راهنمای [موضوع] برای [مخاطب]»
  • «X اشتباه رایج در [موضوع] و راه‌حل‌ها»
  • «چطور [نتیجه] بدون [مشکل رایج]»
  • «[موضوع]: از صفر تا اجرا»
  • «[A] یا [B]؟ — راهنمای انتخاب»

4) مقدمه (حداکثر ۱۲۰ کلمه)​

ساختار:

  1. مشکل/سوال واقعی (Hook)
  2. وعده راه‌حل (Promise)
  3. «در این مقاله...» (Scope)

نکته: کاربر باید در ۱۵ ثانیه اول تصمیم بگیرد بماند. مقدمه باید کوتاه، واضح و مسئله‌محور باشد.

5) Table of Contents (فهرست مطالب)​

  • برای مقالات بالای ۱۵۰۰ کلمه الزامی است
  • لینک‌های Anchor به هر H2
  • در Desktop: Sticky Sidebar
  • در Mobile: Accordion بالای مقاله

6) بدنه مقاله (Body)​

قوانین ساختاری​

  • H2: بخش‌های اصلی (شامل کلیدواژه‌های مرتبط)
  • H3: زیربخش‌ها
  • پاراگراف‌ها: حداکثر ۳–۴ خط
  • Bullet/Numbered List: برای گام‌های اجرایی و مقایسه
  • Bold: فقط برای «نکات کلیدی» (نه تزئینی)

انواع محتوای میانی (Mid-Content Elements)​

  • Call-out Box: نکات مهم / هشدار / تعریف
  • مثال واقعی: «مثلاً یک فروشگاه آنلاین با ۵۰۰ SKU…»
  • Inline CTA: در ۳۰–۴۰٪ ابتدای مقاله (بعد از Hook)
  • تصویر/Diagram: هر ۵۰۰–۷۰۰ کلمه یک المان بصری

ساختار پیشنهادی بدنه بر اساس نوع مقاله​

How-to (راهنمای گام‌به‌گام):

H2: مقدمه / چرا مهم است؟
H2: پیش‌نیازها
H2: گام ۱
H2: گام ۲
H2: ...
H2: اشتباهات رایج
H2: جمع‌بندی + CTA

Comparison (مقایسه):

H2: چرا این مقایسه مهم است؟
H2: معیارهای مقایسه
H2: [A] چیست؟
H2: [B] چیست؟
H2: جدول مقایسه
H2: کدام برای شما مناسب‌تر است؟
H2: جمع‌بندی + CTA

Problem/Solution (مشکل/راه‌حل):

H2: مشکل را بشناسیم
H2: چرا راه‌حل‌های رایج کافی نیستند؟
H2: رویکرد صحیح
H2: مراحل اجرا
H2: نمونه واقعی
H2: جمع‌بندی + CTA

7) Inline CTA (داخل مقاله)​

  • حداکثر ۲ Inline CTA در هر مقاله
  • CTA اول: بعد از ۳۰–۴۰٪ مقاله (مرتبط با موضوع)
  • CTA دوم: قبل از جمع‌بندی

قالب Inline CTA:

[عنوان کوتاه و مرتبط]
[یک جمله توضیح]
[دکمه: «دریافت راهنما / رزرو جلسه / مشاهده دمو»]

8) جمع‌بندی (Conclusion)​

ساختار:

  1. خلاصه ۲–۳ نکته کلیدی
  2. «قدم بعدی» برای کاربر

ممنوع: جمع‌بندی‌های عمومی («امیدوارم مفید بوده باشد») باید: یک توصیه اجرایی مشخص

9) FAQ (اختیاری اما پیشنهاد شده)​

  • برای مقالات بالای ۱۵۰۰ کلمه ۳–۵ سوال
  • پاسخ‌ها کوتاه (۱–۳ جمله)
  • از People Also Ask گوگل برای ایده استفاده شود
  • Schema: FAQPage

10) CTA نهایی (End CTA)​

هر مقاله باید با یکی از این ۳ مسیر تمام شود:

مسیر A — خدمات:

  • CTA: «مشاوره رایگان» یا «درخواست تحلیل»
  • رنگ: Warm Primary (طبق Token-first)

مسیر B — محصول:

  • CTA: «درخواست دمو [نام محصول]»
  • رنگ: Warm Primary

مسیر C — Lead Magnet:

  • CTA: «دریافت [نام Lead Magnet] (رایگان)»
  • رنگ: Warm یا Secondary

11) مقالات مرتبط (Related Articles)​

  • ۳ مقاله مرتبط
  • ترجیحاً از همان Topic Cluster
  • اگر صفحه خدمت/محصول مرتبط دارید، آن هم اضافه شود

12) Schema (پیشنهادی)​

  • Article / BlogPosting (همیشه)
  • BreadcrumbList (همیشه)
  • FAQPage (اگر بخش FAQ دارید)
  • HowTo (اگر مقاله گام‌به‌گام است)
  • Person (نویسنده)

سنجش کیفیت مقاله (QA Checklist)​

سئو​

  • H1 دارد (یک عدد)
  • کلیدواژه اصلی در H1، مقدمه و H2 اول
  • Meta Description نوشته شده (۱۴۰–۱۵۵ کاراکتر)
  • Alt text همه تصاویر
  • حداقل ۲ لینک داخلی به صفحات خدمات/محصولات
  • حداقل ۱ لینک داخلی به مقاله مرتبط دیگر

محتوا​

  • مقدمه زیر ۱۲۰ کلمه
  • پاراگراف‌ها حداکثر ۳–۴ خط
  • حداقل یک مثال واقعی
  • CTA داخل مقاله (Inline)
  • جمع‌بندی با قدم بعدی مشخص
  • CTA نهایی واضح

UI​

  • Featured image آماده (WebP / ۱۲۰۰×۶۳۰)
  • Table of Contents (اگر بالای ۱۵۰۰ کلمه)
  • Mobile خوانا است؟
  • تیتر اصلی روی موبایل قطع نمی‌شود؟
