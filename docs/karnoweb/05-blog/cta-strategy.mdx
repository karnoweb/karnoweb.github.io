# استراتژی CTA بلاگ (Blog CTA Strategy)

## Goal (هدف)

بلاگ بدون CTA = ترافیک بی‌اثر.  
هر مقاله باید کاربر را به یکی از ۳ مسیر ببرد:
1) دریافت Lead Magnet (ورود به قیف)
2) مشاهده خدمت/محصول مرتبط (ارزیابی)
3) رزرو جلسه / درخواست دمو (تبدیل)

---

## قانون اصلی

> «CTA باید با **موضوع مقاله** هم‌راستا باشد — نه یک دکمه عمومی در انتها.»

اگر مقاله درباره «سئو تکنیکال» است:
- CTA مناسب: «دریافت چک‌لیست سئو» یا «تحلیل رایگان سایت»
- CTA نامناسب: «درخواست دمو ERP»

---

## ۴ موقعیت CTA در مقاله

### موقعیت ۱ — Inline (داخل مقاله)
- **کِی:** بعد از ۳۰–۴۰٪ مقاله (بعد از Hook اما قبل از پایان)
- **چه:** Lead Magnet مرتبط
- **سبک:** Call-out Box

```
─────────────────────────────────
📥 [نام Lead Magnet]
[یک جمله توضیح ارزش]
[دکمه: «دریافت رایگان»]
─────────────────────────────────
```

---

### موقعیت ۲ — End of Article (انتهای مقاله)
- **کِی:** بعد از جمع‌بندی، قبل از FAQ و مقالات مرتبط
- **چه:** بسته به مرحله کاربر (Lead Magnet یا خدمت/محصول)
- **سبک:** Card واضح با دو CTA

```
─────────────────────────────────
[عنوان مرتبط با موضوع مقاله]
[یک جمله توضیح]
[CTA اصلی: «دریافت / رزرو / دمو»]
[CTA ثانویه: «مشاهده خدمت/محصول»]
─────────────────────────────────
```

---

### موقعیت ۳ — Sticky Sidebar (Desktop)
- **کِی:** در طول کل مقاله
- **چه:** یک CTA ثابت (Lead Magnet یا مشاوره رایگان)
- **قانون:** بعد از ۳۰۰px اسکرول ظاهر شود، روی TOC نپوشد

---

### موقعیت ۴ — Exit Intent (اختیاری)
- **کِی:** وقتی کاربر قصد خروج دارد
- **چه:** Lead Magnet مرتبط با مقاله
- **قانون سخت:** اگر کاربر یک‌بار رد کرد، دوباره نشان داده نشود.

---

## ماتریس CTA به دسته بلاگ

| دسته | CTA اول (Lead) | CTA دوم (Consideration) | CTA سوم (Conversion) |
|------|---------------|------------------------|----------------------|
| سئو | «دریافت چک‌لیست سئو» | «مشاهده خدمت سئو» | «تحلیل رایگان سایت» |
| طراحی وب | «چک‌لیست ارزیابی سایت» | «مشاهده نمونه‌کارها» | «رزرو جلسه طراحی» |
| ERP | «چک‌لیست انتخاب ERP» | «مشاهده شاپ ERP» | «درخواست دمو ERP» |
| CRM | «قالب مدیریت لید» | «مشاهده CRM کارنووب» | «درخواست دمو CRM» |
| اتوماسیون | «قالب نقشه فرآیند» | «مشاهده خدمت اتوماسیون» | «رزرو جلسه مشاوره» |
| مارکتینگ | «قالب تقویم محتوا» | «مشاهده خدمات مارکتینگ» | «رزرو جلسه استراتژی» |
| هوش مصنوعی | «راهنمای شروع با AI» | «مشاهده خدمات اتوماسیون» | «رزرو جلسه مشاوره» |
| کسب‌وکار | «ماشین‌حساب ROI» | «مشاهده همه خدمات» | «رزرو جلسه استراتژی» |

## رنگ‌بندی CTAها (Token-first)

- **CTA اصلی (Conversion):** Warm Primary
  - `--gradient-warm` + `--shadow-brand-warm`
  - مثال: «رزرو جلسه» / «درخواست دمو»

- **CTA مرتبط با Lead Magnet:** Secondary (Outline) یا Warm
  - اگر هدف اصلی مقاله جمع‌آوری لید است: Warm
  - اگر Soft است: Secondary

- **CTA «مشاهده خدمت/محصول»:** Ghost / Link
  - رنگ: Brand Blue

---

## فرمول‌های متن CTA برای بلاگ

### Lead Magnet
- «دریافت [نام] رایگان»
- «دانلود چک‌لیست (رایگان)»
- «ذخیره راهنما برای بعد»

### Consideration
- «مشاهده نمونه‌کارهای مرتبط»
- «بیشتر درباره [خدمت/محصول] بدانید»
- «ببینید چطور [نتیجه] ممکن است»

### Conversion
- «رزرو جلسه مشاوره رایگان»
- «درخواست تحلیل رایگان سایت»
- «درخواست دمو [محصول] با سناریوی واقعی شما»

---

## قوانین اجرایی CTA بلاگ

| قانون | توضیح |
|------|------|
| حداکثر ۲ Inline CTA | بیشتر = حواس‌پرتی |
| CTA هم‌راستا با موضوع | CTA نامرتبط = کلیک صفر |
| Warm فقط برای Conversion | Lead Magnet می‌تواند Secondary باشد |
| بدون HEX مستقیم | همه از توکن‌های `colors-and-theme.md` |
| Micro Trust زیر CTA | «رایگان» / «بدون تعهد» / «پاسخ زیر ۲۴ ساعت» |

---

## UTM / Tracking (رویدادهای پیشنهادی)

### Query String استاندارد برای CTA بلاگ
- `source=blog`
- `medium=article`
- `content=[article-slug]`
- `topic=[category]`

### مثال
```
/contact?type=consultation&source=blog&medium=article&content=seo-technical-guide&topic=seo
```

### GA4 Events پیشنهادی
- `blog_cta_click`
  - `cta_type`: lead_magnet / service / demo / consultation
  - `article_slug`
  - `position`: inline / end / sidebar / exit

---

## Notes

- CTA بلاگ نباید «یک قالب ثابت» داشته باشد؛ باید متناسب با موضوع و مرحله کاربر تنظیم شود.
- هر ۳ ماه CTAهای پربازدیدترین مقالات از نظر CTR و تبدیل بررسی شوند.
- اگر یک مقاله ترافیک بالا دارد ولی تبدیل صفر، اول CTA را بررسی کنید.
